Challenge Description

In this exercise, we ask that you generate a feed of recommended posts for a user based on their browsing patterns. The tech stack includes Python, Flask, Jinja, HTML, CSS, and Javascript.

Background Info

The feed page in its current state is available here, or can also be accessed by clicking "feed" in the header menu above.

  • The "Latest Posts" feed contains the most recently published posts.
  • The "Trending Posts" feed contains the articles that are currently trending.
  • The "Recommended Posts" feed is empty for now, but will be completed by you during the course of this challenge.

Each of the feeds fetches data from an API in script.js

  • Latest: /api/latest, JSON
  • Trending: /api/trending, JSON
  • Recommended (incomplete): /api/recommended (Note that this feed, when complete, will include posts from trending/latest feeds, but filtered to contain more personalized results. You will likely be editing api.py and main.py files, or adding new files for JSON processing.)

Make note of the data available to you in the existing JSON files. This data can be used for personalization.

Your Task

Using the article information available to you via the APIs, determine the criteria by which you can generate a "Recommended Posts" feed.

For example, you may want to show more "Celebrity" posts if a user tends to click on articles in the "Celebrity" category. Or you may want to give preference to more recently published articles if a user clicks on articles mostly in the "Latest Posts" feed.

Time Constraints

Please do not spend more than 2 hours on this task. At the end you can discuss problems you had with the task, or optimizations you would make if given more time.

Product Requirements

  • Create an API that will return 3 results to populate the "Recommended Posts" feed using your preferred criteria.
  • Use session, local storage, or cookies to store information about the user as they click on articles and navigate the site, which you can then pass along with your API request.
  • Each time the feed page renders, the "Recommended Posts" feed should contain improved recommendations for that user based on their browsing patterns.
  • When you've completed the task, please add some comments to explain:
    • What was your thought process for selecting criteria for personalized recommendations?
    • If given more time, what improvements would you make to the feed page or your recommendations? (e.g. deduplication, SEO, etc)
    • Were any parts of the task particularly challenging or did you require further clarification? We want to improve this task and would love to have your feedback!
